# 프로젝트 버전 관리 행동 규칙 (Workspace Rules)

## 📌 버전 인상 및 배포 주기 속도 조절 룰
1. 자잘한 디자인 피드백, 레이아웃 미세 조정, 배지 문구 수정 등 마이크로 단위의 수정 사항은 \`package.json\` 버전을 매번 올리지 않고 그대로 보존하거나 커밋만 쌓아둡니다.
2. 의미 있는 수준 높은 개선 사항들이 **3~4개 이상 모여** 선생님께서 실서버 테스트를 마치시고 만족하셨을 때, 배포 시점에 비로소 하나의 완성된 기능 단위(예: \`v1.9.0\` 또는 \`v1.8.4\` 등)로 버전을 한꺼번에 통합 판 올림 합니다.
3. 이 규칙은 오답노트 프로젝트에 참여하는 모든 AI 코딩 에이전트에게 예외 없이 엄격하게 강제 적용됩니다.
